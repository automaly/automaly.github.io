01 — The shift

One platform, one admin center

For years, Finance & Operations lived in its own world: managed through Lifecycle Services (LCS), on its own infrastructure, separate from everything else in Dynamics 365. That era is closing.

Under the One Dynamics One Platform strategy, a Finance & Operations environment is now simply an app running on Power Platform, backed by a Dataverse database — sitting right alongside Sales, customer engagement apps, Power Apps, flows, and Copilot agents. You manage all of it from a single place: the Power Platform admin center (PPAC).

One admin. One admin center.

A single portal — plus CLI, PowerShell, connectors, and REST APIs — for low-code apps, Copilot agents, and your ERP. The unified admin experience for F&O apps has reached general availability for new customers.

02 — Convergence

All roads lead to Power Platform

The old model scattered administration across separate portals and consoles. The new model collapses them into one control plane.

03 — The deadline

The freeze that makes it real

This isn't a "someday" roadmap. Microsoft has already drawn the line for new implementations.

A code freeze landed in January 2026 to prepare for the change. Then, starting February 16, 2026, new customers can no longer create cloud implementation projects in LCS for Finance, Supply Chain Management, Project Operations, or Human Resources — they go straight to PPAC. Existing implementations stay on LCS for now, but the direction is unambiguous: PPAC is the destination.

05 — Go live

Go-live just got easier

Previously, you couldn't simply copy a sandbox into production. Now there's a cleaner path: prepare a sandbox with all your software and migrated data, then flip it to production with a single command or API call.

The conversion is generally available, covers the whole environment (Dataverse, customer engagement apps, and the ERP together), and runs with a downtime window of up to about two hours. Plan it, communicate it, then validate with smoke tests once the type reads "Production."

06 — Automation

Built for automation

PPAC is an ERP wrapped in an intelligent control plane. Nearly every admin action is scriptable, so lifecycle management can live inside your CI/CD pipelines.

  • Power Platform API, CLI, PowerShell & connectors — the foundation for management automation across environments.
  • Azure DevOps pipelines — build and release with approvals, and dual-deploy to both LCS-managed and PPAC-managed environments during the transition.
  • Service principal support — deploy with credential-less application users instead of storing passwords, a meaningful security upgrade for ALM.
  • Scoped, granular permissions — delegate exactly what a user or managed identity needs, from reading environment groups to invoking Copilot Studio admin actions.
07 — Before you start

Three prerequisites worth getting right

Security & permissions. Decide who can create environments through tenant settings, and restrict it to administrative roles. Assign service-level admins such as the Dynamics 365 Administrator or Power Platform Administrator.

Licensing. Finance, Supply Chain Management, Project Operations, and Commerce licenses all grant the ability to install the F&O app. Provision using the right ERP template for your implementation — for example a Finance or Supply Chain template.

Storage. The model is capacity-based: deploy what you need, when you need it, and create environments on demand rather than paying monthly for fixed Tier-2 capacity. A minimum of 1 GB of database capacity is required to create or copy an environment.
